# Self-Evaluation

## 1. Хэрэв шалгалт өнөөдөр болбол би энэ кодыг өөрөө бичиж чадах уу?

**Хэсэгчлэн — тайлбартай:**

- **Model класс (Book, Member, Loan):** Тийм — field, getter, setter, constructor-ийг
  өөрөө бичиж чадна. Энэ хэсгийг бүрэн ойлгосон.
- **Repository layer:** Тийм — Java List, stream, Optional ашиглалтыг ойлгосон.
  findByTitle-н contains логикийг өөрөө бичиж чадна.
- **Service layer:** Хэсэгчлэн — бизнес логикийг ойлгосон, гэвч
  LoanService-д BookRepository inject хийх хэсгийг мартаж болзошгүй.
- **Unit test:** Хэсэгчлэн — @BeforeEach, assertThrows-ийг ойлгосон,
  гэвч бүх edge case-ийг өөрөө бодож гаргахад цаг шаардагдана.

## 2. Дахин хийнэ гэвэл юуг өөрөөр хийх вэ?

- **Folder бүтцийг эхэнд зөв төлөвлөх** — давхар src/ folder үүссэнийг
  эхнээсээ болгоомжилж байх байсан.
- **AI session log-ийг яриа явцад нь хадгалах** — сүүлд нь санах ойноос
  бичэх нь алдаатай байж болзошгүй.
- **Commit-ийг илүү олон, жижиг хийх** — feature бүрийн дараа commit хийх байсан.
- **CLAUDE.md-г эхэнд бүрэн бичих** — явцад шинэчлэх нь хэцүү байсан.

## 3. Энэ туршлагаас юу сурсан бэ?

- **AI "verify, don't trust" зарчим практикт юу гэсэн үг болохыг ойлгосон** —
  AI бичсэн код заавал зөв биш, шалгах шаардлагатай.
- **Layered architecture-н ач холбогдлыг ойлгосон** —
  model, repository, service тусдаа байхад тест бичих, засах хялбар болдог.
- **Java Optional-г зөв ашиглах чухалыг ойлгосон** —
  `.get()` шалгалтгүй дуудах нь runtime алдаа үүсгэнэ.
- **AI нь хурдан prototype гаргахад хэрэгтэй, гэвч архитектурын шийдвэрийг
  хүн гаргах ёстой гэдгийг ойлгосон.**
